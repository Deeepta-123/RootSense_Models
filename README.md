The model artifacts you see are the components necessary to run the RootSense soil intelligence pipeline. Here's what each one represents and how they relate to the main models mentioned in your summary:
score_regressor_rf.joblib: This is your primary Soil Health Score model, a Random Forest Regressor (best_rf_reg) that predicts the numerical soil health score from 0-100.
crop_classifier_xgb.joblib: This is your Crop Recommendation model, an XGBoost Classifier (best_xgb_crop) that predicts the most suitable crop.
soil_classifier_rf.joblib: This is your Soil Type Identification model, a Random Forest Classifier (best_rf_soil) that identifies the soil type.
shap_explainer_score.joblib: This is the SHAP TreeExplainer (explainer_score) specifically trained on the Soil Health Score model. It's crucial for the SHAP Explainability feature, allowing you to understand which sensor readings are driving the soil health score up or down.
label_encoder_crop.joblib: This object (le_crop) is used to convert the numerical outputs of the Crop Recommendation model back into human-readable crop names (e.g., turning a model's prediction of 0 into 'Maize').
label_encoder_soil.joblib: Similar to the crop encoder, this object (le_soil) converts the numerical outputs of the Soil Type Identification model back into human-readable soil type names (e.g., turning a model's prediction of 0 into 'Clay').
pipeline_config.json: This JSON file contains all the non-model configuration parameters for the inference pipeline, such as the list of raw and engineered features, the ideal ranges and weights used for calculating the soil health score, and the defined crop and soil type classes. It's the blueprint that guides how the raw sensor data is transformed and interpreted by the models.
